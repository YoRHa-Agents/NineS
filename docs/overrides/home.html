NineS

Multi-Vertex AI Agent Evaluation, Collection, Analysis & Self-Iteration

A self-improving toolkit that evaluates, learns from, and iterates on AI agent capabilities — through 19 measurable dimensions and the MAPIM cycle.

Get Started View on GitHub

Three-Vertex Capability Model

Three interconnected vertices form a self-reinforcing loop — each vertex feeds data to the others, creating emergent intelligence.

V1

Evaluation

Structured evaluations with multiple scoring strategies, matrix evaluation across N axes, and reliability metrics like pass@k.

  • TOML task definitions
  • Exact / Fuzzy / Rubric / Composite scorers
  • Sandboxed execution with pollution detection
  • Markdown & JSON reports
V2

Collection

Discover, fetch, and track external data sources with incremental collection and snapshot-based change detection.

  • GitHub REST + GraphQL
  • arXiv search & metadata
  • SQLite storage with FTS5
  • Token-bucket rate limiting
V3

Analysis

Analyze codebases into structured knowledge through AST parsing, pattern detection, and multi-strategy decomposition.

  • Cyclomatic complexity
  • Cross-file dependency graphs
  • Functional / concern / layer decomposition
  • Knowledge indexing & search

Key Features

⟳

Self-Iteration (MAPIM)

Closed-loop self-improvement across 19 dimensions. Measure → Analyze → Plan → Improve → Measure.

⬡

Sandboxed Execution

Three-layer isolation — process, venv, and tmpdir — with pollution detection across 4 dimensions.

◈

19 Evaluation Dimensions

From scoring accuracy to structure recognition to system-wide convergence rate.

⊞

Extensible Scorers

Protocol-based scoring with registry and entry-point plugin discovery. No inheritance required.

◎

Deterministic Execution

Master seed propagation ensures every evaluation run is exactly reproducible.

⌘

Agent Skill Support

Install NineS as a skill into Cursor or Claude Code with a single nines install command.

Quick Start

Get running in under a minute.

# Clone and install
git clone https://github.com/YoRHa-Agents/NineS.git && cd NineS
uv sync

# Run your first evaluation
uv run nines eval samples/eval/hello_world.toml

# Analyze a codebase
uv run nines analyze ./src/nines --depth standard

# Run self-evaluation across all 19 dimensions
uv run nines self-eval
